# 安全策略

## 适用范围

这个仓库是 HEPH（Hephaestus）的公开代码库。

适合在这里公开反馈的内容包括：

- 可公开讨论的 bug
- 桌面客户端安全问题
- 客户端工作流问题
- 公开代码审查发现

如果问题可能暴露用户数据、凭据或部署敏感行为，不要直接在公开 issue 中贴出完整利用细节。

## 如何报告安全问题

如果你发现的问题可能影响用户数据、凭据或部署敏感行为，请不要公开披露完整利用方式。

建议私下联系项目维护者，并尽量提供：

- 受影响区域
- 影响范围
- 复现步骤
- PoC 或相关证据
- 如果有的话，也可以附上修复建议

## 哪些问题通常可以公开反馈

以下内容通常适合公开报告：

- 桌面体验问题
- 公开仓库中的打包问题
- 文档问题
- 不会影响敏感系统的客户端 bug

## 处理原则

安全相关问题应优先私下确认和收敛，再决定是否适合后续公开说明。
