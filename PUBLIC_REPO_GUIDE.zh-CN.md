# HEPH 公开仓库说明

这个仓库是 HEPH（Hephaestus）的对外核心代码库。

它适合用来做 GitHub 展示、架构讨论、公开 issue 和社区贡献。

## 推荐用途

这个仓库适合承载：

- 桌面壳改进
- 前端交互和体验优化
- 工作节点（miner）工作流讨论
- gateway 路由可读性与实现评审
- 文档和开发体验修复

## 工作模型

HEPH 采用桌面优先、混合推理的模式：

- 本地能跑时优先本地执行
- 需要外部容量时走远程执行
- 根据能力在两者之间自动路由

这就是项目最核心的执行模型，也是这个仓库组织方式的基础。

## 发布建议

对外发布时，直接发布这个目录，而不是更大的私有工作目录。

在推送公开更新前，建议确认：

1. 仓库中只包含源码、文档和你明确想公开的资源
2. 不包含本地日志、缓存、安装包和临时构建产物
3. 环境变量模板仍然只保留占位符
4. 你希望贡献者理解的公开流程已经验证过

## 贡献边界

适合公开贡献的方向包括：

- 桌面打包和启动体验
- 聊天与任务 UI
- 工作节点（miner）工作流质量
- 路由可视化与开发体验
- 文档改进

如果某些部署相关或内部依赖并不在这个公开目录中，那是有意为之；贡献应保持在当前公开仓库边界内。

## 发布心态

把这个公开仓库当作产品对外代码面：

- 文档尽量通用
- 避免机器相关假设
- 用清晰示例代替内部捷径
- 默认贡献者无法看到你的私有工作区上下文
