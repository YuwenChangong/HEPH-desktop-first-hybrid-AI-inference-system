# Security Policy

## Scope

This repository is the public codebase for HEPH (Hephaestus).

Please use this repository for:

- public-safe bug reports
- desktop client security issues
- client-side workflow issues
- public code review findings

Do not use public issues to post exploit details for issues that could expose user data, credentials, or deployment-sensitive behavior.

## Reporting a security issue

If you find a security issue that could affect user data, credentials, or deployment-sensitive behavior, do **not** post the full exploit publicly.

Report it privately to the project owner first with:

- affected area
- impact
- reproduction steps
- proof-of-concept details
- suggested remediation if available

## Safe public reports

These are usually reasonable to report publicly:

- desktop UX issues
- packaging mistakes in the public repository
- documentation mistakes
- client-side bugs that do not expose private systems

## Response expectations

Security-sensitive reports should be handled privately first, then disclosed publicly only after the issue is understood and the risk is contained.
